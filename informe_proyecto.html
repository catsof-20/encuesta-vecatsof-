1. Visión y Concepto de Diseño

Vecatsof se define como una plataforma de captación de clientes de alta gama. Hemos roto con los formularios tradicionales para crear una experiencia basada en el Neo-Glassmorphism, donde la interfaz parece flotar sobre un fondo de gas neón dinámico.

La psicología del color utilizada (pinks y oranges vibrantes) está diseñada para reducir la tasa de abandono en el registro, manteniendo al usuario estimulado visualmente.

2. Anatomía Tecnológica (Stack)

Node.js & Express

Utilizado como el motor de ejecución asíncrono. Express gestiona el enrutamiento y la entrega de la API JSON de forma eficiente y ligera.

MySQL Relacional

Base de datos de nivel empresarial. Gestionamos esquemas vinculados con claves foráneas para asegurar que cada cliente tenga su dirección asociada.

TypeScript (Strict)

Añade una capa de inteligencia al código. Evita errores de tipo "null pointer" y define interfaces estrictas para la comunicación API.

CSS3 & Animations

Uso masivo de variables CSS y animaciones GPU-accelerated para lograr 60fps de fluidez en las transiciones de pasos del formulario.

El flujo del sistema ha sido diseñado para ser atómico y seguro. A continuación se muestra el diagrama de flujo lógico que conecta las tres capas de la aplicación.

4. Auditoría de Seguridad y Calidad

Resumen de Protección

  • Prevención SQLi: Uso de sentencias preparadas en el driver mysql2. Los campos de entrada nunca tocan el motor SQL sin ser saneados automáticamente.
  • Transacciones ACID: Garantizamos la Atomicidad. Si por ejemplo falla el guardado de la calle del usuario, el registro del nombre del cliente se cancela automáticamente para evitar datos corruptos.
  • Tipado Estricto de Datos: El frontend valida los objetos antes de enviarlos, reduciendo la carga de errores en el servidor.

5. Guía de Código Atlas

Ejemplo de la lógica de transacción multi-tabla implementada en el backend:

// server.js - Registro Atómico
db.beginTransaction(err => {
    const sqlClient = 'INSERT INTO clientes (...) VALUES (...)';
    db.query(sqlClient, params, (err, result) => {
        const clienteId = result.insertId;
        const sqlAddress = 'INSERT INTO direcciones (cliente_id, ...) VALUES (?, ...)';
        db.query(sqlAddress, [clienteId, ...], err => {
            db.commit(err => console.log('Transacción Exitosa'));
        });
    });
});
